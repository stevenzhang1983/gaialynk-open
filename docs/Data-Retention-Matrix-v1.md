# 数据保留与删除矩阵（v1）

> **状态**：正式版 v1；保留期为 **占位数值**，标注「待产品 / 法务确认」后可替换为定稿数字（对齐 CTO 指令 E-17、补充规格 §17.3）。  
> **用途**：对内统一归档口径与 `data-retention.job` TTL 默认值；对外与帮助中心、隐私政策一致后再更新用户可见文案。  
> **工程实现**：`packages/server/src/modules/data-retention/data-retention.job.ts`；表字段迁移 `0032_data_retention_archived.sql`。环境变量覆盖：`DATA_RETENTION_<TABLE>_DAYS`（表名大写，如 `DATA_RETENTION_MESSAGES_DAYS`）。

### 2.1 占位脚注（必读出）

- 上表数字为 **可运行默认**，便于实现 `archived` / `archived_at` 软删除作业；**不等同于法务最终承诺**。  
- 定稿流程：产品 + 法务书面确认 → 更新本文件版本号（v1.1…）→ 调整环境变量或代码内默认值。

## 3. 与工程交付的关系

- **已实现**：每日可选作业（`DATA_RETENTION_JOB_ENABLED=1`）按 TTL 将过期行标记 `archived = true` 并写入 `archived_at`。  
- **读路径**：后续 Epic 应在列表/详情查询中默认 `WHERE archived = FALSE`（当前未全局改读路径，避免越权变更范围）。  
- **草案文件**：`docs/Data-Retention-Matrix-Draft-v1.md` 保留历史；以 **本 v1** 为准。
