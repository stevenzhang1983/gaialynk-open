# ADR-0002: 审计 Append-Only 维持与查询能力增强

- 状态：Accepted
- 日期：2026-03-15
- 决策人：主线工程（CTO 授权执行）

## 背景

Phase 0 已有审计事件持久化与分页，但在 Phase 1 需要：

1. 更强的可治理查询能力（按 actor/agent/时间窗口审计）；
2. 保持 append-only 语义，不引入可篡改写路径。

## 决策

1. 审计写路径保持 append-only，不新增 update/delete 能力。
2. 增强审计查询过滤参数：
   - `agent_id`
   - `actor_type`
   - `from`
   - `to`
   - 兼容既有 `event_type / conversation_id / cursor / limit`
3. 接口与存储实现（内存/PG）同步扩展，保证行为一致。

## 结果

- 审计可按责任主体与时间窗口进行精确追踪；
- 无需破坏既有 API 契约，现有调用方无感升级；
- 审计链路“只增不改”原则继续保持。

## 风险与缓解

- 风险：查询参数增多后误用导致性能压力。
  - 缓解：保留分页与 limit，后续按查询热点补索引与基线测试。
- 风险：调用方使用非法参数。
  - 缓解：参数校验与默认行为前置，保持标准错误响应格式。
