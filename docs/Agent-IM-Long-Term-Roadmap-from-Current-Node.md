# Agent IM 長期路線圖（以當前節點為起點）

> 依據：`Agent-IM-Product-Plan.md`（八大赛道、Phase 0～3、12.5 收斂版、12.6 Founder 驗收、12.7 初心回歸重排）  
> 起點：**Staging/生產可部署、主網+官網雙託管、可信調用與審計底座已具雛形、正進入「可驗證上線 + 消費層增量」**  
> 角色：CTO 統籌視角；時間為**相對階段**，可隨數據與資源滾動校準。

---

## 一、當前節點定義（Now）

| 維度 | 狀態 |
|------|------|
| **產品主張** | 普通用戶零門檻「說需求 → 拿結果」；供給質量先於流量放大。 |
| **技術底座** | 主網（會話、目錄、A2A、審計/收據、人審隊列、節點註冊與目錄同步最小閉環等）與官網（Ask、任務、連接器、證據視圖等）已可串聯；身份與 trust token 閉環在工程上已收口。 |
| **部署與運維** | 主網 Railway（或 Render）+ Postgres；官網 Vercel；**不以 Supabase 為戰略依賴**（見 `Agent-IM-CTO-Tech-Stack-and-Infrastructure.md`）。 |
| **產品入口** | 官網區（敘事/轉化）+ 產品區（真實任務）；用戶路徑仍從官網進主網能力。 |
| **並行線** | **平台主線**（協議、主網、可信調用）+ **開源最小發布線**（文檔、SDK、節點接入說明）+ **硬體線後置**（團隊就位後 Phase 1 起接）。 |

**本節點的「完成定義」**：Staging/生產三連門禁可綠、灰度後核心鏈路可觀測；Founder 能在 30 分鐘內用數據做 go/no-go（對齊 12.6.2）。

## 三、H1：驗證與供給（約 0～6 月）

**目標**：在**不返工底座**前提下，證明「用戶因可信與結果質量願意留下」，而非僅協議完成度高。

### 3.1 產品與體驗（對齊 12.7 A/B）

| 優先級 | 內容 | 對應規劃 |
|--------|------|----------|
| P0 | **專業 Agent 供給**：上架標準、首批類目、輕審核、健康檢查與失敗語義（Refinement v1 供給側） | 賽道二、六；12.7-A |
| P0 | **Ask → 路由 → 結果 → Fallback**：自然語言入口、單/雙 Agent 路由、可解釋路徑 L1/L2 | 賽道四、Refinement；12.7-B |
| P0 | **人審與高風險可見**：隊列、收據、審計可追溯 | 賽道三 |
| P1 | **訂閱任務最小閉環**：創建/暫停/恢復/刪除/歷史（與用戶歸屬綁定） | 12.7-C；16 節鋪墊 |
| P1 | **本地連接器 1～2 個高價值動作**：白名單、授權、收據 | 12.7-D；Refinement |

### 3.2 平台與工程

- 主網：遷移穩定、SLO 與告警雛形、契約測試與發布門禁常態化。
- 官網：`Now` / `Coming Soon` 與真實能力一致（12.1A）；漏斗與線索可經營（12.6.1）。
- 開源線：協議文檔與 SDK 與當前 API 對齊；節點接入文檔隨主網演進更新。

### 3.3 驗收與決策

- **Founder 周看板**：`weekly_trusted_invocations`、主路徑漏斗、多 Agent 佔比、高風險攔截準確率、線索質量（13.2.1）。
- **Kill/調整信號**：若連續數週「用戶不因可信調用而選擇我們」，按 13.2.2 收斂敘事或範圍，而非盲目加功能。

## 四、H2：網路與商業化起步（約 6～18 月）

**目標**：從「單託管主網 + 消費閉環」進入「**可連、願連**的網路效應」與「**可收費**的託管與模板」。

### 4.1 Phase 1 核心切片（對齊 12.3、12.5.4）

| 方向 | 要點 |
|------|------|
| **會話體驗** | 線程、@、在線狀態、邀請增強（賽道一） |
| **發現** | 能力標籤、匹配/推薦、私有/公網策略（賽道二） |
| **信任** | 數據邊界、人審升級、可驗證收據落地、聲譽/信任圖 v1、注入檢測 v1（賽道三） |
| **記憶與路由** | 記憶更新、意圖解析與路由（賽道四） |
| **可觀測** | 回放、用量/token 儀表、人可讀導出（賽道五） |
| **連接節點** | 目錄同步 + 跨節點中繼最小閉環、免費額度、連接嚮導、協議兼容承諾（11.3） |
| **消費者一鍵部署** | 模板庫、一鍵實例化、試用額度、成功引導（第十六節） |
| **計費** | 計費模型配置、用量記錄；完整賬單可分階段（賽道六） |

### 4.2 Phase 2 開局（對齊 12.4）

- 編排 DSL/運行時雛形；託管 Agent 運行時與沙箱抽象接軌（11.4）。
- **IM 橋雙軌**：官方 1～2 個 Connector + **Bridge API v1**（第十七節）。
- 連接節點分成與結算設計落地（可與產品節奏分階）。
- **硬體線**（若團隊就位）：Agent Dock Phase 1——綁定、插上在線、設備身份、本地行動中樞與高風險確認（賽道八、9.2.8）。

### 4.3 硬體與執行 Agent（條件啟動）

- 未就位則本地平移；就位後與主網身份、收據、人審聯動優先於「功能堆疊」。

## 五、H3：規模與生態（約 18～36+ 月）

**目標**：主網作為 **Hub** 的網路效應、企業可採購、生態可變現。

| 主題 | 內容（對齊產品規劃） |
|------|----------------------|
| **編排與市場** | 編排模板與市場、分成與結算閉環（賽道四、六） |
| **合規與企業** | 合規報表、預設策略包、白標增強、私有化/專屬雲選項（賽道三、七、十五節） |
| **通道與開放** | 實時音視頻（按需）、多 IM、Webhook、合作夥伴計劃（賽道七） |
| **物理世界與硬體** | 執行 Agent 目錄、southbound 適配層、多設備小隊、空間級協同（第十節、賽道八） |
| **進階信任** | TEE/attestation、加固運行時等按企業與場景漸進（賽道三） |

## 七、戰略原則（貫穿全圖）

1. **用戶結果優先於協議複雜度**（12.7.1）。  
2. **供給先於增長**；公開 Agent 質量與可替換回退機制保口碑（Refinement）。  
3. **可信調用是護城河**：決策可解釋、收據可審計、高風險默認保守（賽道三、4.4）。  
4. **雙區模型長期有效**：官網敘事與產品區能力分層演進（12.1A）。  
5. **基礎設施可換、抽象不換**：Postgres、託管廠商可按階段升級（CTO 技術棧文檔「何時重新審視」）。
