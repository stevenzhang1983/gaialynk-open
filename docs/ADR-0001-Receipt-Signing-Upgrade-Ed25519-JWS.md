# ADR-0001: 收据签名升级到 Ed25519/JWS（保留 HMAC 回退）

- 状态：Accepted
- 日期：2026-03-15
- 决策人：主线工程（CTO 授权执行）

## 背景

Phase 0 使用 HMAC 签名可满足最小闭环，但存在：

1. 对称密钥共享，不利于跨组织验签；
2. 不利于形成长期可演进的签名标准契约。

Phase 1 目标要求“可审计、可治理、可长期演进”，因此需要升级到非对称签名格式。

## 决策

1. 新增 Ed25519/JWS 签名路径作为主路径：
   - Header: `alg=EdDSA`, `typ=JOSE`, `kid=<key id>`
   - Payload: `payload_hash`, `conversation_id`, `audit_event_id`, `issued_at`, `prev_receipt_hash`
2. 保留 HMAC 签名作为兼容回退路径（环境未配置 Ed25519 key 时启用）。
3. 验签逻辑按 `signer` 自动分流：Ed25519/JWS 或 HMAC。

## 结果

- 社区版可继续最小可运行（无密钥时使用 HMAC 回退）。
- 生产部署可切换到 Ed25519/JWS，并支持跨域验签。
- 测试新增 Ed25519/JWS 覆盖，保证升级链路可验证。

## 风险与缓解

- 风险：密钥配置错误导致签名回退到 HMAC。
  - 缓解：通过 `signer` 字段可观测当前签名模式，部署时增加配置检查。
- 风险：签名标准演进兼容性。
  - 缓解：保留 `kid` 并约定密钥轮换策略，后续可增多版本验签。
