# Agent IM OSS vs Cloud 能力矩阵（草案）

日期：2026-03-15  
适用范围：`gaialynk-open` 开源仓库

---

## 1. 分层原则

- **OSS（开源）**：可接入标准、最小可运行能力、可验证闭环。
- **Cloud / Enterprise（闭源）**：规模化运营能力、商业策略、计费结算与企业治理能力。

---

## 2. 能力矩阵

| 能力域 | 社区版（OSS） | Cloud / Enterprise（Closed） | 说明 |
| --- | --- | --- | --- |
| 协议与契约 | A2A 扩展、Node-Hub 最小协议、收据格式 | 协议网关运营策略、多租户隔离策略 | OSS 提供可接入契约 |
| 会话协作 | 会话、消息、目录、最小多 Agent 调用 | 企业级编排 DSL、复杂意图路由策略 | OSS 保持最小协作闭环 |
| 可信调用 | 基础 Trust Policy（allow / need_confirmation / deny）、审计、收据验签 | 高级策略包、行业治理规则、复杂信誉模型 | 规则可解释优先开源 |
| 人审流程 | Review Queue 查询、确认、幂等防重入 | 企业审批流编排、多级审批策略 | OSS 保留最小人审可用性 |
| 节点网络 | 节点注册/心跳/目录同步最小闭环 | 托管运行时调度、容量管理、多区域运营策略 | OSS 优先保障可连可测 |
| 控制台 | 极简演示控制台与验证流程 | 运营后台、企业管理后台、白标能力 | 严格分离 demo 与运营后台 |
| 指标与观测 | 北极星与基础可观测指标（go/no-go） | 商业化增长看板、计费报表、SLA 报表 | OSS 暴露非敏感口径 |
| 计费与商业化 | 无 | 计费、分成、结算、反作弊与风控规则 | 明确闭源，不进入 OSS |

- `docs`: **OPEN-OK**（需持续保持“社区版 vs Cloud”边界说明）

---

## 4. 争议项与处理

1. **收据签名实现细节是否开源**  
   结论：开源基础签名与验签流程（可验证），不内置生产密钥与运营策略（已采用环境变量注入）。

2. **go/no-go 指标是否会泄露运营策略**  
   结论：保留基础口径用于验证，不暴露商业化阈值体系与增长运营策略实现细节。
