# Agent IM 主线团队 Phase 1 工作规划（下发版）

日期：2026-03-15  
下发对象：主线开发团队（平台后端 / 协议 / 基础控制台）  
对齐依据：`Agent-IM-Product-Plan.md` 第 `12.1`、`12.3`、`12.5.4`

---

## 1. 执行定位（必须统一）

当前项目已从 Phase 0 进入 **Phase 1（体验与发现）**。  
主线团队本阶段职责是：把“可演示闭环”升级为“可持续使用产品能力”，不是扩展 Phase 2 功能。

---

## 2. Phase 1 主线必须完成项（来自 12.5.4）

1. 会话体验补强：线程、@、在线状态、邀请流程增强  
2. 连接节点可用化：目录同步 + 跨节点中继最小闭环、兼容承诺、连接向导、免费额度支撑  
3. 赛道三关键增强：数据边界策略、人审工作流升级、可验证收据正式落地、链下声誉 v1、注入检测与告警/拦截 v1  
4. 一键部署首个 Agent 的主线支撑能力（模板实例化所需接口与治理约束）

---

## 3. 接下来 6 周执行节奏（3 个双周迭代）

## Iteration A（W1-W2）：会话与信任可用化

### 目标

让核心协作路径在真实任务中稳定可用。

### 交付

- 线程/@/在线状态关键路径稳定
- 高风险确认与人审链路优化（幂等、防重入、审计一致）
- 数据边界策略最小可行版（消息入会话与请求转发前执行）

### DoD

- 首次会话成功率较基线提升
- 高风险路径零误放行
- 审计/收据/决策链路可完整追溯

## Iteration B（W3-W4）：节点可用化与互操作

### 目标

把“可连”升级为“愿连、可协作”。

### 交付

- 节点目录同步 + 跨节点中继稳定化
- `Node-Hub` 兼容性承诺文档与版本策略
- 连接向导所需后端契约与错误码稳定化

### DoD

- 新节点可在标准流程内接入成功并参与会话
- 关键异常（离线/重试/不匹配）有稳定行为与测试覆盖

## Iteration C（W5-W6）：一键部署支撑与阶段收口

### 目标

支持“首个 Agent 一键部署”的后端最小闭环，并形成阶段验收资产。

### 交付

- 模板实例化所需主线接口与约束
- 试用额度/升级引导所需计数与限流接口
- Phase 1 中期验收包（契约、测试、发布、联调证据）

### DoD

- 非技术用户路径可从入口走到首次会话（后端无阻塞）
- Founder 30 分钟可完成中期 go/adjust 决策

---

## 4. 与官网团队协作边界（强约束）

- 主线只提供稳定、脱敏、只读或明确写路径接口，不接受临时高耦合需求
- 官网团队不得改动主线可信调用核心决策逻辑
- 联调按固定窗口进行，输出问题单并分级（阻塞/高优先/可延后）

---

## 5. 本阶段禁止项

- 禁止提前做完整编排 DSL
- 禁止提前做完整计费账单系统
- 禁止提前做全量 IM Connector
- 禁止为了赶进度放松契约门禁和收据/审计完整性

---

## 6. 每周固定交付（主线）

1. 契约变化报告（含影响声明）  
2. 高风险路径回归结果  
3. 节点可用化指标快照  
4. 与官网团队联调结论与问题单  

---

## 7. 一句话执行原则

**Phase 1 主线的任务不是“新增最多功能”，而是“让可信调用协作网络在真实使用中稳定成立”。**
