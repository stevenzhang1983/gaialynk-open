# Agent IM 官网/入口团队 Phase 1 工作规划（下发版）

日期：2026-03-15  
下发对象：官网与入口团队（Web / 内容 / 增长）  
对齐依据：`Agent-IM-Product-Plan.md` 第 `12.1A`、`12.3`、`13.2`

---

## 1. 执行定位（必须统一）

官网团队在 Phase 1 的角色是：  
**把“官网区（Marketing Zone）-> 产品区（Product Zone）”的用户路径做成可持续转化系统。**

不是目标：

- 不是把官网做成产品控制台
- 不是追求视觉炫技而牺牲转化与数据质量
- 不是临时驱动主线开发高耦合接口

---

## 2. 关系口径（来自 12.1A，必须执行）

1. 官网区负责讲清价值、场景、文档、CTA  
2. 产品区负责真实使用（会话、目录、风险确认、审计、收据）  
3. 用户路径固定为：**先官网教育，再进入产品区完成任务**  

---

## 3. 接下来 6 周执行节奏（3 个双周迭代）

## Iteration A（W1-W2）：入口转化链路稳定化

### 目标

让三语入口和主漏斗稳定、可追踪、可复盘。

### 交付

- 三语关键页面（EN/繁中/简中）口径一致与路由稳定
- 主漏斗固定：`home -> start_building -> docs_click -> activation`
- 线索链路稳定：`book_demo` / `join_waitlist` 提交可追踪可导出

### DoD

- 三语关键入口均可用且无断链
- 主漏斗与次漏斗事件口径一致
- 线索数据可查询、可导出、可复盘

## Iteration B（W3-W4）：数据质量与反滥用强化

### 目标

确保增长数据可信，不被噪音带偏决策。

### 交付

- 可疑流量识别、告警、历史追溯稳定
- 导出任务与告警历史 TTL 清理策略执行
- 周复盘机制固定化（告警 -> 动作 -> 回收）

### DoD

- 可疑流量占比可解释并受控
- 连续两周复盘有完整闭环记录
- 关键指标可支撑 Founder 周决策三问

## Iteration C（W5-W6）：三语优化与实验放大

### 目标

降低 locale gap，提升有效激活与线索质量。

### 交付

- 针对 `zh-Hant` / `zh-Hans` 的定向优化实验（至少 1 轮）
- 页面承诺一致性检查并纳入固定流程
- 实验报告（假设/阈值/护栏/决策）标准化

### DoD

- 至少 1 条关键漏斗指标有可验证改善
- 实验结论可明确给出：保留/回滚/继续迭代

---

## 4. 与主线协作边界（强约束）

- 官网仅消费主线稳定、脱敏、只读接口
- 不推动主线临时开发非阶段目标接口
- 所有页面承诺必须与主线已交付能力一致

---

## 5. 本阶段禁止项

- 禁止把官网区做成控制台交互承载层
- 禁止为追求 CTR 弱化数据质量约束
- 禁止提前引入重 BI 体系拖慢迭代
- 禁止脱离 `12.1A` 关系设定改信息架构

---

## 6. 每周固定交付（官网）

1. `Weekly Review` 复盘文档（含 owner/due/status）  
2. 漏斗与告警快照  
3. 实验卡片（假设/样本窗口/成功阈值/护栏）  
4. 页面承诺一致性检查结果  

---

## 7. 一句话执行原则

**官网团队的 Phase 1 目标是“把入口做成可信、可复盘、可持续优化的转化系统”，而不是“做成更漂亮的网站”。**
